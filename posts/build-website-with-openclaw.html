使用 OpenClaw 构建个人网站与部署 AI 工具完整指南

2025年3月10日 阅读时间约 15 分钟
返回文章列表

本文详细记录了如何使用 OpenClaw 从零开始构建一个现代化的个人网站，并将 GitHub 项目部署为可在线使用的工具。整个过程展示了 AI 辅助开发的强大能力。

第一部分：个人网站构建

1.1 项目背景

目标：创建一个类似现代 AI 产品官网（Vercel/Notion 风格）的个人网站

设计目标：

  • 极简、现代、科技感
  • 深色主题
  • 清晰的信息结构
  • 强调 AI、自动化、效率工具主题
  • 提升视觉层级和可读性

1.2 技术栈选择

  • 前端：纯 HTML5 + CSS3 + JavaScript（无框架依赖）
  • 样式：Tailwind CSS 风格自定义 CSS
  • 字体：Inter（Google Fonts）
  • 部署：Nginx + 腾讯云服务器
  • 版本控制：Git + GitHub

1.3 网站结构

网站架构：
├── Navbar（固定导航栏）
│   ├── Logo
│   └── 导航链接：首页、博客、工具、实验、关于
│
├── Hero Section（首屏）
│   ├── 左侧：标题 + 副标题 + CTA按钮
│   └── 右侧：代码终端视觉效果
│
├── Exploration Section（探索领域）
│   ├── AI Tools
│   ├── Automation
│   └── Productivity
│
├── Tools Section（工具展示）
│   └── SmartCS-V3（AI客服冷启动工具）
│
├── Blog Section（博客文章）
│   └── OpenClaw 新手入门教程
│
├── About Section（关于我）
│   └── 个人介绍 + 联系方式
│
├── Subscribe Section（订阅）
│   └── 邮箱订阅表单
│
└── Footer（页脚）
    └── 版权信息 + 社交链接

1.4 设计规范

颜色系统：

--bg-primary: #0f172a      /* 主背景 */
--bg-secondary: #1e293b    /* 次级背景 */
--text-primary: #f8fafc    /* 主文字 */
--text-secondary: #94a3b8  /* 次级文字 */
--accent-primary: #6366f1  /* 强调色 */
--accent-secondary: #8b5cf6 /* 渐变结束 */

视觉效果：

  • 渐变背景（紫色到蓝色）
  • 毛玻璃效果（backdrop-filter）
  • 卡片悬停动画（scale + shadow）
  • 滚动渐入动画

1.5 响应式设计

断点设置：

  • 桌面端：> 1024px
  • 平板端：768px - 1024px
  • 移动端：< 768px

第二部分：SmartCS-V3 工具集成

2.1 项目介绍

SmartCS-V3 是一个 AI 客服冷启动工具，基于 DeepSeek API 构建。

功能特性：

  • 上传客服对话记录（CSV/TXT）
  • AI 自动分析业务场景
  • 生成标准作业程序（SOP）
  • 识别高频问题和解决方案

技术栈：

  • React 19
  • TypeScript
  • Vite
  • DeepSeek API
  • Tailwind CSS

2.2 部署挑战与解决方案

挑战 1：CORS 跨域限制

问题：浏览器无法直接调用 DeepSeek API

解决方案：Nginx 反向代理

location /api/deepseek {
    proxy_pass https://api.deepseek.com/chat/completions;
    proxy_set_header Authorization "Bearer $DEEPSEEK_API_KEY";
    proxy_set_header Content-Type "application/json";
    proxy_set_header Host api.deepseek.com;
}

挑战 2：API Key 安全存储

问题：Key 硬编码在代码中存在泄露风险

解决方案：

  1. 将 Key 存储在服务器安全目录：/etc/nginx/secrets/
  2. 设置文件权限为 600（仅 root 可读）
  3. 通过环境变量或后端服务读取

挑战 3：访问控制与限流

问题：公开的 API 可能被滥用

解决方案：Node.js 后端服务

// 速率限制：每分钟 20 次
const limiter = rateLimit({
  windowMs: 60 * 1000,
  max: 20,
  message: { error: '请求过于频繁' }
});

// 密码验证
const authMiddleware = (req, res, next) => {
  const { password } = req.body;
  if (password !== 'limuzi2025') {
    return res.status(401).json({ error: '访问被拒绝' });
  }
